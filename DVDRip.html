Übersicht

DVDRip.ps1 liest eine eingelegte DVD mit MakeMKV aus und komprimiert die erzeugten MKV-Dateien anschließend mit HandBrakeCLI. Optional werden Fortschrittsnachrichten per Telegram versandt und die DVD nach Abschluss automatisch ausgeworfen.

Parameter

Konfig-Datei

ParameterStandardBeschreibung
-Config–Pfad zu einer JSON-Konfig-Datei. Alle dort definierten Werte werden als Standardwerte übernommen. Explizit per CLI übergebene Parameter haben stets Vorrang.

Verzeichnisse & Ausgabe

ParameterStandardBeschreibung
-OutputDir Pflicht*–Zielverzeichnis. Darunter wird automatisch ein Unterordner pro DVD angelegt. Kann alternativ über die Konfig-Datei angegeben werden.
-DiscTitleautoName des DVD-Unterordners. Wird automatisch aus dem Disc-Inhalt ermittelt, falls nicht angegeben.
-LogDirOutputDirBasisverzeichnis für Log-Dateien. Wird als Basis verwendet wenn -LogFile einen relativen Pfad enthält.
-LogFile–Dateiname oder Pfad der Log-Datei (Append-Modus). Absoluter Pfad wird direkt verwendet; relativer Pfad wird mit -LogDir (bzw. OutputDir) kombiniert. Ohne Angabe wird ein neues Timestamped-File in OutputDir angelegt.
-TempDir%TEMP%Lokales Zwischenverzeichnis für Rip und Encoding bei Netzwerk-OutputDir.

MakeMKV

ParameterStandardBeschreibung
-DriveIndex0MakeMKV-Laufwerksindex. Einzelner Wert oder kommagetrennte Liste für mehrere Laufwerke (-DriveIndex 0,1). Bei mehreren Laufwerken werden diese sequenziell verarbeitet. Im Service-Modus werden alle angegebenen Laufwerke überwacht.
-MinLength1800Minimale Titellänge in Sekunden. Kürzere Titel werden übersprungen.
-MinMkvSizeMB500Minimale MKV-Dateigröße in MB. Bricht MakeMKV mit einem Fehler ab, aber die größte erzeugte Datei ist mindestens so groß, wird der Fehler ignoriert und HandBrakeCLI trotzdem gestartet. Typisch bei Lesefehlern am Ende der Disc.
-MakeMKVPathautoPfad zu makemkvcon.exe. Wird automatisch erkannt falls leer.

HandBrakeCLI

ParameterStandardBeschreibung
-HBPresetHQ 576p25 SurroundHandBrakeCLI-Preset (ab v1.11). Leer = kein Preset. Ausgabe-Container wird automatisch aus dem Preset-Namen abgeleitet (MKV wenn Name „MKV" enthält, sonst MP4).
-HBQuality22RF-Qualitätsfaktor. Kleinere Werte = bessere Qualität, größere Datei.
-HBExtraArgs–Zusätzliche HandBrakeCLI-Argumente als einzelner String.
-HandBrakePathsiehe obenPfad zu HandBrakeCLI.exe.

Schalter

ParameterBeschreibung
-DebugMode SwitchAlle Log-Zeilen farbig auf Stdout ausgeben. HandBrake-Fortschritt wird live gestreamt.
-SkipRip SwitchMakeMKV-Schritt überspringen. Nützlich wenn MKV-Dateien bereits vorhanden sind.
-SkipEncode SwitchHandBrakeCLI-Schritt überspringen. Nur rippen, nicht kodieren.
-NoEject SwitchDVD nach Abschluss nicht auswerfen. Standard: DVD wird automatisch ausgeworfen.
-ServiceMode SwitchDauerbetrieb: Das Skript läuft kontinuierlich und wartet nach jeder verarbeiteten DVD automatisch auf die nächste Disc (WMI-Event + Polling). Ideal für den Betrieb als Windows-Dienst.

Service-Modus

ParameterStandardBeschreibung
-PollInterval5Polling-Intervall in Minuten, mit dem im Service-Modus auf eine neue DVD geprüft wird. Nur relevant bei aktiver -ServiceMode-Option.

Telegram

ParameterBeschreibung
-TelegramTokenBot-Token. Leer = keine Benachrichtigungen.
-TelegramChatIdChat-ID des Empfängers.

Benachrichtigungen werden gesendet bei: Start, Rip-abgeschlossen, Kodierung-abgeschlossen, Fehler und Fertig ("Bitte nächste DVD einlegen").

Beispiele

Mit Konfig-Datei

# Alle Einstellungen aus DVDRip.json laden
.\DVDRip.ps1 -Config ".\DVDRip.json"

# Konfig als Basis, einzelne Parameter per CLI überschreiben
.\DVDRip.ps1 -Config ".\DVDRip.json" -DiscTitle "MeinFilm" -SkipRip

Minimal (alles automatisch)

.\DVDRip.ps1 -OutputDir "D:\Videos" -DebugMode

Mit Preset und Qualität

.\DVDRip.ps1 -OutputDir "D:\Videos" -HBPreset "H.265 MKV 1080p" -HBQuality 20

Nur rippen, nicht kodieren

.\DVDRip.ps1 -OutputDir "D:\Videos" -SkipEncode -DebugMode

Nur kodieren (MKV bereits vorhanden)

.\DVDRip.ps1 -OutputDir "D:\Videos" -DiscTitle "MeinFilm" -SkipRip -DebugMode

Ohne automatischen Auswurf

.\DVDRip.ps1 -OutputDir "D:\Videos" -NoEject

Zwei Laufwerke gleichzeitig befüllen

# Einzel-Modus: Laufwerk 0 dann Laufwerk 1 sequenziell verarbeiten
.\DVDRip.ps1 -OutputDir "D:\Videos" -DriveIndex 0,1

# Service-Modus: beide Laufwerke überwachen
.\DVDRip.ps1 -OutputDir "D:\Videos" -DriveIndex 0,1 -ServiceMode

Als Dienst im Dauerbetrieb

# Einmal starten – läuft nach jeder DVD weiter und wartet auf die nächste
.\DVDRip.ps1 -OutputDir "D:\Videos" -ServiceMode

# Polling alle 2 Minuten statt der Standard-5-Minuten
.\DVDRip.ps1 -OutputDir "D:\Videos" -ServiceMode -PollInterval 2

Verzeichnisstruktur

Logging

  • Log-Datei: <LogDir>/DVDRip_<Timestamp>.log
  • Ohne -DebugMode: keine Stdout-Ausgabe außer der finalen Erfolgsmeldung
  • Mit -DebugMode: alle Log-Zeilen farbig auf Stdout, HandBrake-Fortschritt live
  • Das Log wird am Ende zusätzlich ins OutputDir kopiert

Exit-Codes

CodeBedeutung
0Erfolg
1Voraussetzungen fehlen (MakeMKV oder HandBrakeCLI nicht gefunden)
2MakeMKV-Fehler oder keine MKV-Dateien erzeugt
3HandBrakeCLI-Fehler
99Unerwarteter Fehler

Ablauf

  1. Voraussetzungen prüfen (MakeMKV, HandBrakeCLI)
  2. Disc-Titel und Laufwerksbuchstaben via MakeMKV info ermitteln
  3. Ziel- und Rip-Verzeichnisse anlegen
  4. Rippen: MakeMKV liest alle Titel ≥ MinLength als MKV in RAW/
  5. Kodieren: HandBrakeCLI komprimiert jede MKV-Datei in den Disc-Ordner
  6. DVD auswerfen (sofern nicht -NoEject)
  7. Telegram-Abschlussbenachrichtigung senden
  8. Log ins OutputDir kopieren
  9. Bei -ServiceMode: auf nächste DVD warten und Vorgang wiederholen

Konfig-Datei

Mit -Config kann eine JSON-Datei angegeben werden, die Standardwerte für alle Parameter enthält. Das ermöglicht saubere Wrapper-Skripte ohne hartcodierte Credentials. Explizit per CLI übergebene Parameter überschreiben den jeweiligen Konfig-Wert. OutputDir kann wahlweise per CLI oder in der Konfig gesetzt werden — fehlt es an beiden Stellen, bricht das Skript mit einem Fehler ab.

Auto-Erkennung: Wird -Config nicht angegeben und werden keine weiteren Parameter übergeben, sucht das Skript automatisch nach einer Datei namens DVDRip.json im aktuellen Arbeitsverzeichnis. Wird sie gefunden, wird sie als Konfig-Datei verwendet. Sobald mindestens ein Parameter explizit angegeben wird, entfällt die Auto-Erkennung — die Config muss dann bei Bedarf explizit mit -Config angegeben werden.

Musterdatei DVDRip.json:

{
    "OutputDir":      "C:\\Daten\\DVDs",
    "TelegramToken":  "DEIN_BOT_TOKEN",
    "TelegramChatId": "DEINE_CHAT_ID",
    "ServiceMode":    true,
    "PollInterval":   2,
    "DebugMode":      true
}

Alle Parameter aus der Tabelle können in der JSON-Datei verwendet werden. Switch-Parameter (DebugMode, ServiceMode etc.) werden als true / false angegeben. Nicht aufgeführte Parameter behalten ihre Standardwerte.

Service-Modus

Mit -ServiceMode läuft das Skript nach dem Verarbeiten einer DVD nicht aus, sondern wartet automatisch auf die nächste eingelegte Disc. Die Erkennung erfolgt zweistufig: primär über einen WMI-Event (sofortige Reaktion auf Laufwerks- änderungen), als Fallback zusätzlich durch regelmäßiges Polling im Abstand von -PollInterval Minuten (Standard: 5).

Dieser Modus eignet sich besonders für den unbeaufsichtigten Dauerbetrieb, z. B. als Windows-Dienst via Install-DVDRipService.ps1. Telegram-Benachrichtigungen werden dabei nach jeder verarbeiteten DVD gesendet, inklusive der Aufforderung, die nächste Disc einzulegen.

ParameterStandardBeschreibung
-ServiceMode SwitchfalseDauerbetrieb aktivieren. Skript wartet nach jeder DVD auf die nächste (WMI-Event + Polling).
-PollInterval5Polling-Intervall in Minuten. Bestimmt, wie oft das Laufwerk zusätzlich zum WMI-Event abgefragt wird.
